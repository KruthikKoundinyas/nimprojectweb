The Game of Nim

Nim is a two-player mathematical strategy game. Players take turns removing objects from distinct heaps. On each turn, a player must remove at least one object from a single heap.

Why Study Nim?

Nim is one of the foundational games in combinatorial game theory. It's simple enough to solve completely, yet rich enough to demonstrate key concepts used in AI, economics, and computer science:

  • Binary arithmetic and XOR operations
  • Game state analysis and optimal decision-making
  • Reinforcement learning in discrete environments
  • The difference between analytical and learned intelligence

The Nim Sum (XOR)

The key to solving Nim is the Nim Sum — the bitwise XOR of all heap sizes. This single number tells you whether you're in a winning or losing position.

How XOR Works

XOR (exclusive or) compares bits column by column. If an odd number of 1s appear in a column, the result is 1. Otherwise, it's 0.

ABA XOR B
000
011
101
110

Step-by-Step: Computing 1 XOR 3 XOR 5

Convert to binary:

1 = 001

3 = 011

5 = 101

XOR column by column:

Column 1 (rightmost): 1 XOR 1 XOR 1 = 1

Column 2: 0 XOR 1 XOR 0 = 1

Column 3: 0 XOR 0 XOR 1 = 1

Result: 111 = 7

Winning Strategy

The mathematical solution to Nim is elegant: always leave your opponent with a Nim Sum of zero.

Winning Position

Nim Sum ≠ 0

You can always find a move that leaves the opponent with Nim Sum = 0

Losing Position

Nim Sum = 0

Any move you make will create a non-zero Nim Sum for your opponent

Finding the Optimal Move

  1. Calculate the Nim Sum of all heaps
  2. Find a heap where heap XOR nimSum < heap
  3. Reduce that heap to heap XOR nimSum
  4. This guarantees the new Nim Sum = 0

Worked Example

Board: [3, 5, 7]

Nim Sum: 3 XOR 5 XOR 7 = 011 XOR 101 XOR 111 = 001 = 1

Check Heap 1 (3): 3 XOR 1 = 2 < 3 ✓

Optimal move: Remove 1 from Heap 1 (3 → 2)

New board: [2, 5, 7] → Nim Sum = 2 XOR 5 XOR 7 = 0 ✓

Why Does This Work?

XOR has a key property: A XOR A = 0. When you reduce a heap so that the total XOR becomes zero, your opponent's any move will necessarily break the zero — creating a non-zero XOR that you can again reduce to zero. You maintain this invariant until the game ends.

The Misere Twist

In Misere Nim, the last player to move loses. This changes the endgame strategy but not the mid-game.

Misere Strategy

Mid-Game (heaps > 1 exist)

Play exactly like Normal Nim — maintain Nim Sum = 0 for your opponent.

As long as multiple large heaps exist, the strategies are identical.

Endgame Switch

When all heaps are 0 or 1, leave an odd number of size-1 heaps.

This forces your opponent to take the last stone.

Why the Switch Works

Consider a position with only size-1 heaps. Each player must take exactly one stone per turn. If there's an odd number of heaps, the first player to move will eventually face the last stone. By leaving an odd number, you ensure your opponent is that player.

Endgame Example

Board: [0, 1, 1, 1] (three size-1 heaps)

Your opponent takes one → [0, 0, 1, 1]

You take one → [0, 0, 0, 1]

Opponent must take the last stone → they lose!

The Critical Transition

The trickiest part is recognizing when to switch. If exactly one heap has size > 1, you must decide how much to leave in that heap. The rule:

  • Count the number of size-1 heaps
  • If that count is even: reduce the large heap to 1 (making an odd total)
  • If that count is odd: reduce the large heap to 0 (keeping an odd total)

Q-Learning Agent

Q-Learning is a model-free reinforcement learning algorithm. The agent learns which moves are good by playing thousands of games and tracking outcomes — without being told the rules of strategy.

Core Concepts

State (s)

The current board configuration, e.g. [1, 3, 5]

Action (a)

A legal move: remove N stones from heap H

Q-Value Q(s,a)

Estimated "quality" of taking action a in state s

Reward (r)

+1 for winning, -1 for losing, 0 otherwise

The Update Rule

Q(s, a) ← Q(s, a) + α [ r + γ · max Q(s', a') − Q(s, a) ]
α = learning rate (0.1) γ = discount factor (0.9) r = immediate reward

Exploration vs Exploitation

The agent uses an ε-greedy policy:

  • With probability ε: explore (choose a random move)
  • With probability 1-ε: exploit (choose the best known move)

ε starts high (lots of exploration) and decays over training episodes, so the agent gradually shifts from exploration to exploitation as it becomes more confident.

Training Process

How NimLab's Q-Agent Trains

1. Initialize Q-values to 0 for all state-action pairs

2. Play 500+ games against an opponent that uses optimal strategy 70% of the time

3. After each game, propagate rewards backward through the moves

4. The agent gradually learns which moves lead to wins

5. Q-values are stored in localStorage and persist between sessions

Why Use Q-Learning When XOR Is Perfect?

The XOR strategy computes the optimal move analytically — it's perfect and instant. So why bother with Q-Learning?

  • Learning from experience — Q-Learning discovers the same policy without being told the rules
  • Generalizable — the same algorithm works on games without known mathematical solutions
  • Demonstrates RL fundamentals — state-action values, temporal difference learning, exploration
  • Comparison value — two fundamentally different approaches converging on the same answer

The educational value is in seeing that experience-based learning can approximate analytical perfection — and understanding when and why it falls short.
